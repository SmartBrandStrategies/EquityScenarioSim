# Code of Conduct

## Our Commitment

We are committed to providing a welcoming and inspiring community for all. We pledge that everyone participating in the Cap Table Simulator community will be treated with respect and dignity.

## Our Standards

Examples of behavior that creates a positive environment include:

- **Being respectful**: Treat all community members with courtesy and respect
- **Being inclusive**: Welcome people of all backgrounds, experiences, and perspectives
- **Being constructive**: Provide helpful feedback and focus on solutions
- **Being collaborative**: Work together to improve the project
- **Listening actively**: Take time to understand others' viewpoints
- **Assuming good intent**: Give others the benefit of the doubt
- **Celebrating others**: Acknowledge and appreciate contributions

Examples of unacceptable behavior include:

- Harassment or discrimination of any kind
- Insulting or demeaning comments
- Personal attacks
- Trolling or inflammatory comments
- Unwelcome sexual attention or advances
- Publishing private information without consent
- Any form of bullying or intimidation

## Scope

This Code of Conduct applies to:
- GitHub issues and pull requests
- Code reviews and comments
- Discussions and forums
- Offline events and meetups
- Any space representing the Cap Table Simulator community

## Enforcement

### Reporting Issues

If you experience or witness unacceptable behavior:

1. **Document the incident** - Note what happened, when, and who was involved
2. **Report privately** - Email maintainers with details
3. **Be specific** - Include examples and context
4. **Respect privacy** - Share only what's necessary

### Maintainer Responsibilities

Maintainers will:
- Take all reports seriously
- Investigate fairly and confidentially
- Respect the privacy of all parties
- Take appropriate action
- Communicate outcomes when appropriate

### Consequences

Violations may result in:
- Warning and request to modify behavior
- Temporary suspension from community
- Permanent ban from project
- Removal of content

## Guidelines for Discussions

### When Submitting Issues
- Use clear, descriptive titles
- Provide sufficient context
- Be respectful to those responding
- Don't cross-post to multiple channels

### When Making Pull Requests
- Write descriptive commit messages
- Reference related issues
- Accept constructive feedback gracefully
- Work collaboratively with reviewers

### When Commenting
- Be constructive and helpful
- Provide evidence for claims
- Acknowledge good ideas
- Avoid dismissive language

## Inclusive Language

We strive to use inclusive language:
- Avoid gendered pronouns when unnecessary
- Be mindful of cultural references
- Use accessible terminology
- Welcome all experience levels

## Getting Help

- **Questions about the project?** Open a discussion
- **Need clarification?** Ask respectfully
- **Unsure about guidelines?** Reach out to maintainers
- **Concerned about behavior?** Report confidentially

## Appreciation

We appreciate:
- Bug reports and feature suggestions
- Code contributions
- Documentation improvements
- Community support and encouragement
- Diverse perspectives and ideas

## Acknowledgment

This Code of Conduct is inspired by the Contributor Covenant and other community standards, adapted for the Cap Table Simulator project.

## Changes to This Code

This Code of Conduct may be updated as the community grows. Changes will be communicated to all members.
